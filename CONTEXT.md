# zettlink

YouTube 영상과 GitHub 저장소를 캡처하고, LLM으로 요약·인사이트를 생성해 학습 산출물로 발전시키는 1인 지식 관리 시스템.

## Language

**소스**:
카드의 기반이 되는 외부 실체. YouTube 영상 또는 GitHub 저장소.
_Avoid_: 원본, 외부 소스

**카드**:
소스 1개를 캡처한 결과로 vault에 생성되는 폴더+파일 묶음. 소스가 삭제되어도 카드는 vault에 영속한다.
_Avoid_: 소스, 원본

**원문**:
소스에서 추출한 원어(영어) 콘텐츠. YouTube는 `transcript.md`, GitHub는 `extract.md`.
_Avoid_: 원본

**캡처**:
사용자가 Telegram에 소스 URL을 전송하는 단일 트리거 행위.
_Avoid_: 아카이빙, 저장, 입력

**아카이빙**:
캡처로 시작되어 추출 → 요약 → vault 저장까지 완료되는 시스템 프로세스. 소스를 영속적으로 보존하는 것이 목적.
_Avoid_: 캡처, 처리

**요약본**:
아카이빙 완료 시 자동 생성되는 카드의 기본 LLM 출력. `index.md`에 저장되며 카드의 얼굴 역할을 한다.
_Avoid_: 산출물, 처리본

**산출물**:
사용자가 대시보드에서 명시적으로 요청해 생성되는 파생 콘텐츠. 심화·TIL·가이드 세 종류가 있다.
_Avoid_: 요약본, 결과물

**심화**:
소스를 타임라인(YouTube) 또는 문맥·섹션(GitHub) 단위로 분해한 장문 분석. 어려운 개념에는 해설을 붙이고, 전체 주장·근거·한계를 비판적으로 검토한다. 요약본보다 현저히 길다.
_Avoid_: 요약, 분석, 심화 요약

**TIL**:
소스를 통해 새롭게 알게 된 것을 1인칭 학습자 관점으로 서술한 짧은 글. 개인적이고 가벼운 톤. 공개 블로그에 게시되는 것을 전제로 작성된다.
_Avoid_: 요약, 정리, 리뷰

**가이드**:
소스의 내용을 본인의 현재 프로젝트에 적용하는 방법을 단계별로 서술한 실용 문서. 코드 스니펫과 명령어를 포함한다.
_Avoid_: 튜토리얼, 요약, TIL

**발행**:
카드 또는 산출물을 공개 블로그에 노출하는 행위. 카드의 `published: true` 플래그로 요약본이 공개되고, 각 산출물(`deep.md`, `til.md`, `guide.md`)은 개별 `published: true` 플래그로 선택적으로 공개된다.
_Avoid_: 배포, 공개, 업로드

**플랫폼**:
소스가 속한 외부 서비스. `youtube` 또는 `github`. frontmatter의 `platform` 필드에 기록된다.
_Avoid_: 채널, 소스 유형, 타입

## Relationships

- **소스** 1개는 정확히 **카드** 1개에 대응된다
- **카드**는 **원문**을 포함한다
- **원문**은 번역하지 않고 영어 그대로 보존된다
- **캡처**는 **아카이빙**의 시작점이다. 캡처 없이 아카이빙은 일어나지 않는다
- **아카이빙**이 완료되면 **카드**가 생성된다. 카드는 반드시 **원문**과 **요약본**을 포함한다
- **산출물**은 카드 생성 이후 사용자가 선택적으로 추가한다. 없는 카드도 유효하다
- **카드**를 발행하면 **요약본**이 공개된다. **산출물**은 각자 별도로 발행한다
- **TIL**은 공개 독자를 전제로 쓰고, **가이드**는 본인 사용을 전제로 쓴다. 발행 여부는 다를 수 있다

## Example dialogue

> **Dev:** "사용자가 YouTube URL을 Telegram에 보내면 뭐가 생기죠?"
> **Domain expert:** "**캡처**가 발생하고 시스템이 **아카이빙**을 시작합니다. 자막을 추출해 **원문**으로 저장하고 LLM으로 **요약본**을 만들어 **카드**가 완성됩니다."
>
> **Dev:** "TIL은 그때 같이 만들어지나요?"
> **Domain expert:** "아닙니다. **TIL**은 자동으로 생성되지 않습니다. 아카이빙 후에 사용자가 대시보드에서 직접 요청해야 만들어지는 **산출물**입니다. **심화**·**TIL**·**가이드** 모두 마찬가지입니다."
>
> **Dev:** "심화가 있는 카드를 발행하면 심화도 같이 공개되나요?"
> **Domain expert:** "아닙니다. **카드**를 **발행**하면 **요약본**만 공개됩니다. **심화**를 공개하려면 `deep.md`를 별도로 발행해야 합니다. **산출물**은 각자 독립적으로 발행합니다."

## Flagged ambiguities

- "원본"은 (1) 소스, (2) 원문 파일, (3) 원본 언어를 동시에 가리켜 충돌 발생 — 해소: **소스** / **원문** / **원어 보존**으로 분리.
