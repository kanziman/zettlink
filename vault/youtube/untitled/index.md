# 모델보다 중요한건 하니스 입니다.

> https://www.youtube.com/watch?v=8DySHAuAmts

## 요약

AI 모델의 성능보다 중요한 것은 모델을 감싸는 '하니스(harness)' 시스템이다. 스킬 기반 리팩토링, 도메인 매뉴얼, 자동 검사 등 모델 주변의 시스템을 잘 구축하면 동일한 모델로도 46점에서 80점까지 성능을 높일 수 있다. 저자는 이 깨달음으로 회사를 나가 AI 개발의 진정한 가치를 추구하게 됐다.

## 인사이트

- 모델은 신입의 IQ이고 하니스는 그 신입을 둘러싼 회사의 시스템
- 하니스 설계로 결과물 품질이 28%에서 47%까지 향상되지만 프롬프트만 다듬으면 3% 미만 상승
- 같은 클로드 모델도 주변 시스템에 따라 벤치마크 점수가 46점에서 80점까지 34점 차이 발생
- 스펙 리뷰, 워크플로우, 자동 검사 등의 하니스로 대규모 코드 마이그레이션을 반나절에 완료
- 하니스는 작업 명세, 컨텍스트 선택, 도구 접근, 검증, 실패 분석 등 11가지 요소로 구성
- 지속적 성공을 위해서는 스킬 기반 리팩토링과 도메인 기반 유지보수의 이중 구조 필요
