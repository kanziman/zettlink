# Anthropic 엔지니어가 마크다운 버린 5가지 이유 — Karpathy "이건 AI 진화 법칙"

> https://www.youtube.com/watch?v=CLI95vLlkxY

## 요약

Anthropic Claude Code 팀 엔지니어 Thariq Shihipar가 마크다운 대신 HTML 사용을 선언했고, OpenAI 공동창업자 Karpathy가 이를 단순한 취향이 아닌 AI 출력의 진화 법칙으로 정의했습니다. 마크다운은 100줄을 넘으면 읽기 어렵고 시각적 표현이 제한적이지만, HTML은 색상, 도표, 인터랙션, 애니메이션 등 풍부한 정보 전달이 가능합니다. 사람의 뇌는 시각 정보 처리에 30%를 할당하므로 HTML을 통한 시각화는 정보 습득 효율을 극대화합니다.

## 인사이트

- 마크다운은 AI 시대 이전의 포맷으로, 100줄 이상 텍스트는 사용자가 읽기 거부
- HTML은 색, 그래픽, 레이아웃, 양방향 인터랙션을 모두 지원해 정보 밀도 극대화
- 인간 뇌는 시각 처리에 30% 할당 - 텍스트만 제공하는 것은 1차선 국도, HTML은 4차선 고속도로
- Claude Code의 MCP 도구 연결로 데이터를 HTML 대시보드로 통합 시각화 가능
- AI 출력의 진화 단계: 텍스트 → 마크다운 → HTML → 인터랙티브 신경 비디오로 진행 중
- 사용자는 HTML 생성을 위해 복잡한 스킬 학습 불필요 - AI에게 '한 줄'로 요청만 하면 됨
